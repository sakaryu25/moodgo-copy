# MoodGo 検索マッピング（実API検証済み版）

> 気分選択.docx の更新版。**Yahoo業種コード（gc）だけは docx の番号が全面的に誤り**（実Yahoo APIで別ジャンルを指す）だったため、**実APIで1コードずつ逆引き検証して確定した正しいコード**に置き換えた版です。
> Supabaseタグ・検索キーワード・「Googleは最終手段」は docx どおりで正しい。
>
> - 検証日: 2026-07-18（本番 `YAHOO_LOCAL_SEARCH_API_KEY` で全48コードをライブ確認）
> - コード上の実体: `app/api/recommend/route.ts` の `DIVE_YAHOO_GC`（約4831行）＝**唯一の正**
> - 検索優先順位: **① Supabase（自前DB）→ ② Yahoo（無料）→ ③ Google（最終手段）**。Google補足検索は `skipYahooOnly` の時だけ発火する保険。

## ⚠️ 最重要警告 — docxの業種コードに戻さないこと
docx の番号は実APIでは全く別のジャンルを指す。戻すと検索が壊れる（実測）:

## 実装メモ（開発者向け）
- **唯一の正**は `app/api/recommend/route.ts` の `DIVE_YAHOO_GC`（deep-dive文字列 → gcコード）。この表はその写し＋実API確認結果。コードを変えたらこの表も更新すること。
- Yahoo検索は **①キーワード検索（1語ずつ分割・AND回避）** と **②業種コード検索（キーワード無し）** の2系統を実施してマージ（`route.ts` 参照）。
- gc欄 `—` は「Yahooに適切な業種コードが無い」→ キーワード検索のみ（展望台・夜景・道の駅・図書館・食べ歩き・つくる体験など）。
- 検索優先: **Supabase → Yahoo(無料) → Google(最終手段・`skipYahooOnly`時のみ)**。
- 全48コードは 2026-07-18 に本番APIキーでライブ検証済み（`scratchpad/yahoo-gc-all.mjs`）。
